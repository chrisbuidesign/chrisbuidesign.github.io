GrooveMachine

Role

UI/UX Lead

Timeline

3 months

Format

Web

Context

I led the design efforts by conducting comparative research, developing lo-fi wireframes, and creating hi-fi mockups.

GrooveMachine is a tangible interface that uses music to teach young students about coding concepts. Tangible pieces can be placed onto its table surface. The center octagonal hub serves as a step sequencer, which causes the tangibles to play a music sample when their section is passed over by light.

An interactive webpage was created to digitize the physical experience.

Goals & Challenges

Redesign GrooveMachine's website without drastically changing existing interactions.

Physical interface

Approaching Redesign

While a broswer version of GrooveMachine already existed, I needed to figure out how to make it fresh without undoing the hard work that was already put into it. It led me to some questions regarding my first moves:

  • What functions are users already familiar with?
  • How closely does the website need to mimic the physical interface?
  • What are the pain points of the existing webpage?

Old website

Identifying Key Features and Opportunities

Since both the physical interface and the previous website contained certain features, it was important to make sure that they appeared in this version of the website as well. Critical parts of the original designs include:

  • the step sequencer
  • tangible drawers
  • volume and tempo controls
  • the save function
  • audio visualization
  • code visualization

However, there were some functions missing from either (or both) versions of GrooceMachine such as a lack of:

  • onboarding
  • sequencer directional controls
  • toggles for visuals
  • options for different audio samples

A Fresh Face

With these notes in mind, I began my work by exploring different layout and developing mood boards.

Trials of different layouts

Audio Visualization

The audio visualization distinguished GrooveMachine's web version from its physical version. It was a tricky task to create visuals that were visual interesting but not too distracting.

Audio vis experiments + sketch these out with iPad

Code Visualization

A neat feature of the physical interface was its code visualization. Whenever a user placed a tangible piece on the table, code would appear on a computer screen translating their real-life actions into code. It was updated every time a change was made.

Computer w/ code vis

Although it wasn't in the previous web app, it was decided that it would be featured in this new one to get users intrigued about code.

Code vis in UI

New Interaction Opportunities

A bottom menu was added to include the "Direction" and "Sample Pack" since they were a crucial part of the table. "Direction" changes the direction of the step sequencer, while "Sample Pack" changes the genre of sounds that are played (i.e. Mariachi, Acoustic, Hip Hop, etc.).

Bottom menu pic

Onboarding was another important feature that needed to be designed. It was balancing act between being informative and being overwhelming.

Onboarding pic

What's Next?

I'd love to get some user feedback on the new web app. I'm also curious if there are other projects out there like GrooveMachine. A comparative analysis would certainly help me make improvements.
